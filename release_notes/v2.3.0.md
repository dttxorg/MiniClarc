# MiniClarc v2.3.0

### Added

- Usage Endpoint: a multi-provider adapter for rate-limit data. Pick Anthropic (default OAuth), MiniMax, OpenAI (preset), or Custom. Each provider has a "Test Endpoint" probe in Settings that shows the HTTP status, raw JSON, and parsed 5h/7d utilization.
- Chat: Codex-style phase cards. When Claude's streamed text contains a `task_update` JSON or XML block, a per-phase card now appears in the assistant message — with a live ticking duration while running, a frozen duration when done, a chevron to expand/collapse the details, and a Changed Files + Tests section. Cards with the same id mutate in place so the start time is preserved across `running` → `done` transitions.
- Chat: Multi-select questions from Claude now show checkboxes and a Submit button so you can pick more than one option. Previously, tapping any option immediately submitted it as the only answer.

### Changed

- Hard fork cleanup: the `main` branch now tracks the `feat/auto-approve-in-project-root` history. The legacy upstream tracking is gone.
- `RateLimitService` rewritten to delegate to a `UsageAdapter` factory; OAuth/Keychain handling is unchanged for Anthropic.
- `MessageBlock` extended with an optional `taskUpdate: TaskUpdateMessage?` field. Old messages without the field decode unchanged.

### Fixed

- `CustomAdapter` now throws `UsageError.missingField` when a user-supplied JSON path can't be resolved, instead of silently returning 0. (The Custom provider requires the user to type a path; silent zero was confusing.)
- MiniMax endpoint pre-fill: switching to MiniMax from a clean state now actually fills the official `https://www.minimaxi.com/v1/token_plan/remains` URL. Previously the pre-fill check was looking at the empty-string case that the AppState normalizer rewrites to `nil`, so it never fired.
