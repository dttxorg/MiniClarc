# Build & Upload MiniClarc v2.3.0 (local on your Xcode Mac)

The release page already exists at https://github.com/dttxorg/MiniClarc/releases/tag/v2.3.0 with the release notes. You just need to attach the binary.

## 1. Pull and verify

```bash
cd <your-local-MiniClarc-folder>
git pull origin main --ff-only
git log --oneline -1
# expect: bc908b2 chore(release): v2.3.0 ...
```

## 2. Make sure your Apple Developer ID is in the keychain

```bash
security find-identity -p codesigning -v
# expect at least one "Developer ID Application: ..." line
```

If nothing shows up, open Xcode → Settings → Accounts → your Apple ID → "Manage Certificates" → "+" → "Developer ID Application". The project already has `DEVELOPMENT_TEAM = 27WJDMWS6V` hard-coded, so a team-matching cert will be picked up automatically.

## 3. Build the .app

```bash
xcodebuild \
    -project Clarc.xcodeproj \
    -scheme Clarc \
    -configuration Release \
    -destination "generic/platform=macOS" \
    -archivePath build/Clarc.xcarchive \
    CODE_SIGN_STYLE=Automatic \
    DEVELOPMENT_TEAM=27WJDMWS6V \
    archive
```

If `archive` succeeds, you'll have `build/Clarc.xcarchive`.

## 4. Export the signed .app

```bash
xcodebuild -exportArchive \
    -archivePath build/Clarc.xcarchive \
    -exportOptionsPlist scripts/ExportOptions.plist \
    -exportPath build/export
```

Result: `build/export/Clarc.app`.

## 5. Notarize (required for distribution)

```bash
# One-time setup
cat > scripts/.env <<'EOF'
APPLE_ID=you@example.com
APP_PASSWORD=xxxx-xxxx-xxxx-xxxx
TEAM_ID=27WJDMWS6V
EOF

# Submit
xcrun notarytool submit build/export/Clarc.app \
    --apple-id "$(grep APPLE_ID scripts/.env | cut -d= -f2)" \
    --password  "$(grep APP_PASSWORD scripts/.env | cut -d= -f2)" \
    --team-id   "$(grep TEAM_ID scripts/.env | cut -d= -f2)" \
    --wait

# Staple
xcrun stapler staple build/export/Clarc.app
```

## 6. Zip the .app (with the metadata flags from build_zip.sh)

```bash
ditto -c -k --keepParent --norsrc --noextattr --noqtn \
    build/export/Clarc.app \
    build/Clarc-2.3.0.zip
```

The `--norsrc --noextattr --noqtn` flags are critical — they stop the zip from including AppleDouble entries that would break Sparkle's code seal.

## 7. Attach to the release

```bash
gh release upload v2.3.0 build/Clarc-2.3.0.zip \
    --repo dttxorg/MiniClarc \
    --clobber
```

Verify at https://github.com/dttxorg/MiniClarc/releases/tag/v2.3.0 — the zip should show up under "Assets".

---

## Alternative: skip notarization, ship ad-hoc

If you only want the zip for personal use on your own Mac (no Gatekeeper, no public distribution), step 5 is optional. You can do everything else as-is and the app will work after right-click → Open the first time.

For a public release though, notarize. It's free, just slow.
