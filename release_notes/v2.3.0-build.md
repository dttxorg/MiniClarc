# Build & Upload MiniClarc v2.3.0 (local on your Xcode Mac)

The release page is at https://github.com/dttxorg/MiniClarc/releases/tag/v2.3.0.
The current attached zip is **`Clarc-2.3.0-arm64.zip`** (3.9 MB, arm64-only, ad-hoc signed).

## 1. Pull and verify

```bash
cd <your-local-MiniClarc-folder>
git pull origin main --ff-only
git log --oneline -1
# expect: 4111eb2 fix(appstate): move migrateUsageProvider() after stored properties init
# (or a later commit that includes the Sparkle-removal + Picker fix)
```

## 2. Make sure Xcode is fully installed

```bash
xcode-select -p
# expect: /Applications/Xcode.app/Contents/Developer (NOT /Library/Developer/CommandLineTools)
```

If it points at CommandLineTools: open the App Store, install Xcode, then:

```bash
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
```

**First-time Xcode 26.x note:** SwiftTerm builds a Metal shader. The Metal toolchain is downloaded separately:

```bash
xcodebuild -downloadComponent MetalToolchain
# 687 MB download, one-time
```

## 3. Build the .app (arm64-only, single architecture)

```bash
rm -rf build && mkdir -p build
xcodebuild \
    -project Clarc.xcodeproj \
    -scheme Clarc \
    -configuration Release \
    -destination "generic/platform=macOS" \
    -derivedDataPath build/DerivedData \
    ARCHS=arm64 ONLY_ACTIVE_ARCH=NO \
    CODE_SIGNING_ALLOWED=NO CODE_SIGN_IDENTITY="" \
    build
```

Why arm64-only:
- The previous fat binary was 29 MB (x86_64 + arm64) and shipped as 8.7 MB zip.
- The arm64-only thin binary is 14 MB; after `strip -x -S` it's **5.7 MB**, app total **9.3 MB**, zip **3.9 MB**.
- Apple Silicon has been the only sensible target for years. If you need Intel support later, change `ARCHS=arm64` to `ARCHS="arm64 x86_64"` (no `ONLY_ACTIVE_ARCH=NO`).

## 4. Ad-hoc sign + strip

```bash
codesign --force --deep --sign - build/DerivedData/Build/Products/Release/Clarc.app
strip -x -S build/DerivedData/Build/Products/Release/Clarc.app/Contents/MacOS/Clarc
codesign --force --deep --sign - build/DerivedData/Build/Products/Release/Clarc.app
```

`strip -x -S` removes the Swift type/metadata symbols that aren't needed at runtime, saving ~8 MB. The ad-hoc re-sign after strip is required because strip invalidates the signature.

## 5. Zip the .app

```bash
ditto -c -k --keepParent --norsrc --noextattr --noqtn \
    build/DerivedData/Build/Products/Release/Clarc.app \
    build/Clarc-2.3.0-arm64.zip
```

The `--norsrc --noextattr --noqtn` flags stop the zip from including AppleDouble entries that would break Sparkle's code seal. (We're not using Sparkle in this fork, but the flags don't hurt.)

## 6. Attach to the release

```bash
gh release upload v2.3.0 build/Clarc-2.3.0-arm64.zip \
    --repo dttxorg/MiniClarc \
    --clobber
```

Verify at https://github.com/dttxorg/MiniClarc/releases/tag/v2.3.0.

## Optional: full Developer-ID-signed build (proper distribution)

If you ever set up a Developer ID Application certificate:

```bash
cat > scripts/.env <<'EOF'
APPLE_ID=you@example.com
APP_PASSWORD=xxxx-xxxx-xxxx-xxxx
TEAM_ID=27WJDMWS6V
EOF

xcodebuild \
    -project Clarc.xcodeproj \
    -scheme Clarc \
    -configuration Release \
    -archivePath build/Clarc.xcarchive \
    CODE_SIGN_STYLE=Automatic \
    DEVELOPMENT_TEAM=27WJDMWS6V \
    archive

xcodebuild -exportArchive \
    -archivePath build/Clarc.xcarchive \
    -exportOptionsPlist scripts/ExportOptions.plist \
    -exportPath build/export

xcrun notarytool submit build/export/Clarc.app \
    --apple-id "$(grep APPLE_ID scripts/.env | cut -d= -f2)" \
    --password  "$(grep APP_PASSWORD scripts/.env | cut -d= -f2)" \
    --team-id   "$(grep TEAM_ID scripts/.env | cut -d= -f2)" \
    --wait

xcrun stapler staple build/export/Clarc.app

ditto -c -k --keepParent --norsrc --noextattr --noqtn \
    build/export/Clarc.app build/Clarc-2.3.0.zip
```

The signed-and-notarized build does **not** need the user's "right-click → Open" dance.
